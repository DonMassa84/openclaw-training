
# Prompt-to-Code Stabilization

## Prinzip

Magische Agenten-Workflows sind am Anfang promptbasiert. Sobald ein Workflow stabil funktioniert, soll er in Code übersetzt werden.

## Warum

```text
Prompts sind flexibel, aber unvorhersehbar.
Code ist dumm, aber stabil.
```

## Vorgehen

```text
1. Workflow manuell/promptbasiert testen
2. erfolgreiche Schritte identifizieren
3. wiederkehrende Schritte in Bash/Python übersetzen
4. Cron/systemd/n8n einbinden
5. Logs erzeugen
6. Fehlerfälle definieren
7. Runbook aktualisieren
```
