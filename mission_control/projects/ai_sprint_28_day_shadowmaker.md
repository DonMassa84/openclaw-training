
# 28-Day AI Sprint – Shadowmaker Variante

## Ziel

In 28 Tagen aus dem stabilisierten Stack ein produktives persönliches AI-Betriebssystem machen.

## Woche 1 – Stabilität

```text
Tag 1: Stack Healthcheck finalisieren
Tag 2: Mission Control prüfen
Tag 3: Memory-Dateien erweitern
Tag 4: Discord-Webhooks optional setzen
Tag 5: Git-Commit-Routine etablieren
Tag 6: ShadowOps-Failed-Services prüfen
Tag 7: Wochenreview
```

## Woche 2 – Projekt-Pipeline

```text
Tag 8: Daily Delight Build definieren
Tag 9: Personal Digest Pipeline skizzieren
Tag 10: n8n Daily Digest vorbereiten
Tag 11: Flowise Workflow-Idee dokumentieren
Tag 12: Telegram-Report erweitern
Tag 13: Projektlog automatisieren
Tag 14: Wochenreview
```

## Woche 3 – Multi-Agent-System

```text
Tag 15: Scout-Rolle definieren
Tag 16: Analyst-Rolle definieren
Tag 17: Builder-Rolle definieren
Tag 18: Archivist-Rolle definieren
Tag 19: Watchdog-Rolle definieren
Tag 20: Finance-Rolle definieren
Tag 21: Rollenreview
```

## Woche 4 – Stabilisierung in Code

```text
Tag 22: Wiederholbare Prompts identifizieren
Tag 23: Prompt-to-Code Kandidat auswählen
Tag 24: Bash/Python-Skript bauen
Tag 25: systemd/n8n Scheduling prüfen
Tag 26: Runbook ergänzen
Tag 27: Git Commit + Backup
Tag 28: Abschlussreview
```

## Erfolgsdefinition

```text
1. CrashBandicot stabil
2. OpenClaw intern OK
3. Flowise OK
4. n8n OK
5. täglicher Healthcheck läuft
6. Mission Control dokumentiert
7. mindestens ein Workflow prompt-to-code stabilisiert
```
