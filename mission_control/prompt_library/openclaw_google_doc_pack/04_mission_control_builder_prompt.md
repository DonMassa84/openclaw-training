# Mission Control Builder Prompt

## Zweck

Erstellt einen Projektbrief für ein Mission-Control-Dashboard für OpenClaw.

## Zielbild

Eine lokale Next.js-App auf localhost, die zeigt und kontrollierbar macht, was der Agent tut.

## Mögliche Screens

- Tasks
- Calendar
- Projects
- Memory
- Docs
- Team
- Visual Office

## Pflichtfragen

1. Welche Screens sind jetzt am wichtigsten?
2. Wie sieht die Agent Crew aus?
3. Was ist die Mission?
4. Welcher visuelle Stil?
5. Welche Integrationen an Tag 1?

## Projektbrief muss enthalten

- Projektname
- Was gebaut wird
- Warum es wichtig ist
- Screens zuerst
- Agent Crew und Mission
- Visual Direction
- Integrationen
- echte OpenClaw-Daten statt Mockdaten
- Tech Stack: Next.js App Router, Tailwind, shadcn/ui
- Prozess: Workspace audit, Fragen, Plan, Approval, Phase 1, Feedback

## Lokale Shadowmaker-Entscheidung

Mission Control darf echte lokale Daten lesen, aber keine Secrets und keine produktiven Änderungen ohne Freigabe ausführen.
