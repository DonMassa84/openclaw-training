
Prompt-to-Code Stabilisierung
Ziel

Wiederkehrende Prompt-Workflows werden erst dokumentiert, dann als kleine stabile Skripte umgesetzt.

Kriterien

Ein Workflow wird in Code übersetzt, wenn:

er mindestens 3× wiederholt wurde
er klare Eingaben/Ausgaben hat
er kein sudo benötigt
er keine Secrets offenlegt
er testbar ist
ein Rollback möglich ist
Standardformat
Zweck
Eingaben
Verarbeitung
Ausgabe
Fehlerfälle
Test
Rollback
Kandidaten
Daily Brief
Freeze Report
Discord Routing
