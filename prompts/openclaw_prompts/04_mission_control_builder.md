# Mission Control Builder

Ziel:
Eine lokale Mission-Control-Oberfläche für OpenClaw aufbauen.

Screens:
1. Tasks
2. Calendar
3. Projects
4. Memory
5. Docs
6. Team
7. Visual Office

Shadowmaker-Priorität:
Phase 1:
- Daily Briefs
- Freeze Reports
- Systemstatus
- Projekte
- Memory

Phase 2:
- Tasks
- Docs
- Team

Phase 3:
- Visual Office
- Kalender
- Agenten-Cadence

Technik:
Next.js App Router, Tailwind, shadcn/ui, localhost, filesystem-first, optional SQLite.

Wichtig:
Keine Mock-Daten. Mission Control muss echte Daten aus dem OpenClaw-/Shadowmaker-Workspace lesen.
